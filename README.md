## 功能

| 功能 | 说明 |
|------|------|
| **手动追踪** | 在类目弹窗中点击任意类目行即可记录/取消 |
| **全量扫描** | 一键采集当前弹窗中所有已加载的类目（含层级关系） |
| **分级显示** | 表格按「一级类目 / 二级类目 / 三级类目 / 类目名称」分列展示 |
| **下载 CSV** | 导出 UTF-8-BOM 编码表格，Excel / WPS 打开中文不乱码 |
| **下载 Markdown** | 导出 `.md` 文档，方便复制到笔记或文档中 |
| **调试诊断** | 开启调试模式后，控制台输出详细日志，方便定位提取问题 |

## 安装

### 前置条件

- Chrome / Edge / Firefox 浏览器
- 安装 [Tampermonkey](https://www.tampermonkey.net/) 扩展

### 安装脚本

1. 点击本页顶部的 **「安装脚本」** 绿色按钮
2. Tampermonkey 会弹出安装确认页面，点击 **「安装」**
3. 登录 [Ozon 卖家后台](https://seller.ozon.ru/)，页面右下角会出现蓝色悬浮按钮

> 也可以手动安装：复制 `ozon-category-collector.user.js` 的内容 → Tampermonkey 图标 → 添加新脚本 → 粘贴保存

## 使用方法

1. 在 Ozon 卖家后台，点击 **「类目」** 按钮打开类目筛选弹窗
2. 采集器自动检测到类目树（状态栏显示「就绪 ✓」）
3. **方式一：手动点击**
   - 在弹窗类目树中 **点击** 任意类目行（复选框或文字区域均可）
   - 点击一次 = **记录**，再点击一次 = **取消**
4. **方式二：全量扫描**
   - 展开你想采集的类目后，点击面板上的 **「全量扫描」** 按钮
   - 自动采集当前弹窗中所有可见类目，并保留层级关系
5. 采集结果实时显示在面板表格中
6. 点击 **「CSV」** 或 **「MD」** 按钮导出

## 面板说明

- 面板可 **拖拽** 移动，不遮挡后台操作
- 点击标题栏 **「▾」** 收起面板
- 面板右下角显示当前追踪数量（橙色角标）

## 导出字段

| 列名 | 说明 | 示例 |
|------|------|------|
| 一级类目 | 根级类目名称 | `爱好和创作` |
| 二级类目 | 二级子类目 | `创造收纳袋和包` |
| 三级类目 | 三级子类目 | `帆布包` |
| 类目名称 | 当前点击的类目名称 | `帆布包` |

> 如果某级不存在（如只有两级），对应列留空

## 技术实现（v5.0）

Ozon 卖家后台的类目树使用动态生成的 CSS 类名，传统方案（基于 class 匹配或缩进像素计算）全部失效。v5.0 的核心改进：

1. **识别 CSS 自定义属性 `--level`**：Ozon 在行级 `<button>` 元素上通过 `style="--level: N"` 标记层级深度，脚本直接读取该值
2. **精准定位行元素**：通过 `button[style*="--level"]` 找到每个类目行，向上回溯到 `li` 行容器
3. **从 button 内部提取名称**：遍历 button 子元素，排除 checkbox 区，找到最宽的文本元素
4. **层级栈算法**：根据 `--level` 值维护一个路径栈，自动构建 L1 > L2 > L3 层级关系

## 故障排查

如果采集结果为空或名称不准确：

1. 确保打开了 **「类目」** 筛选弹窗
2. 点击 **「检测树」** 按钮，确认状态栏显示 `已检测到 ✓ (N 个行)`
3. 开启 **「调试模式」** 复选框
4. 按 `F12` 打开浏览器开发者工具 → **Console**
5. 点击类目弹窗中的元素，控制台会输出详细日志
6. 将日志截图提交 [Issue](https://github.com/vision-png/ozon.seller/issues)
